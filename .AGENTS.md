## 1. Project at a Glance

**Kerdos** is a privacy-first React Native (Expo) app for Greek freelancers and small businesses. It tracks income, expenses, and profit distributions and calculates net profit, income tax, and tax prepayment (Προκαταβολή Φόρου) in real-time. All data is stored locally in SQLite — no accounts, no servers.

- **Language:** TypeScript + React Native
- **Router:** expo-router (file-based)
- **Database:** expo-sqlite (SQLite, local only)
- **Localization:** i18next — English + Greek (Ελληνικά)
- **State:** React Context (`AppSettingsContext`)
- **Icons:** expo-symbols (SF Symbols — iOS native strings)
- **Haptics:** expo-haptics

## 4. Critical Rules — Never Violate

### Database
- **Always add new schema columns via `ALTER TABLE ... ADD COLUMN` in `initSchema()` wrapped in `try/catch`** — never a fresh `CREATE TABLE` for existing tables.
- **Always update `lib/backup.ts`** when a column is added: update the row interface AND the restore INSERT, using `?? ''` fallback for backward-compat with old backups.
- Never write raw SQL in components or screens — all DB logic lives in `lib/database.ts`.

### Localization
- **Every user-facing string must have a key in both `en` and `el`.** The `el` object is typed as `typeof en`, so TypeScript will catch missing keys at compile time.
- Never hardcode user-visible strings in JSX.
- Brand names (OpenAI, Claude Code, Apple Developer, EFKA, etc.) stay in English in both languages.

### Theming
- **Never hardcode colors** like `#ffffff` or `#000000` in components — always use `colors.card`, `colors.text`, etc. from `useAppColors()`.
- Exception: entry type accent colors (`#22c55e` income, `#ef4444` expense, `#a855f7` distribution) are hardcoded by convention.
- Always use the `useStyles(colors)` pattern — styles in `useMemo(() => StyleSheet.create({...}), [colors])`.

### UX Patterns
- Long press on entry rows: **600ms** (`delayLongPress={600}`) + `Haptics.impactAsync(Medium)`.
- Amount validation: `parseFloat(amount.replace(',', '.'))` — always accept comma as decimal separator.
- Save flow: validate → setSaving(true) → try/catch DB call → success alert → callback → finally setSaving(false).

## 7. Adding a New Expense Category — Step-by-Step

This is the most common small task, documented here as a worked example:

1. **`lib/types.ts`** — add the new ID to the `ExpenseCategoryId` union type
2. **`lib/types.ts`** — add the entry to `EXPENSE_CATEGORIES` array (before `other`): `{ id, label, icon, color }`
   - `icon` must be a valid SF Symbol string (use `SFSymbol` type — TypeScript will validate)
   - `color` should be visually distinct from existing category colors (see `features.md` table)
3. **`lib/i18n.ts`** — add `yourId: 'Label'` to `categories` in **both** `en` and `el` objects

That's it — the category automatically appears in the expense picker, entry list, expense breakdown, and recent additions chips.

## 8. Current App State Summary

- **Supported languages:** English, Greek
- **Supported themes:** Light, Dark, Device
- **Entry types:** Income (with client name), Expense (18 categories), Profit Distribution (with shareholder name)
- **Edit flow:** Long press any entry row on the dashboard (600ms) → bottom-sheet modal
- **Recent Additions:** Last 5 unique entries per type shown as tappable chips on the Add screen
- **Backup format:** JSON v1, file: `kerdos-backup-YYYY-MM-DD.json`
- **Tax defaults:** 22% income tax, prepayment disabled by default
